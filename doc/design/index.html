RESTful Web Service design doc

The system is divided in 3 basic units:

  • Places
  • Vehicles
  • Connections
By using the GET method is possible to retives the list of the respectevely elements. To manage the great possible amount of data, a paging system has been implemented. Each lists can be accesses by dividing them in pages, this can improve performance. Is also possible to accessing to a selected page directly.

Single place can be accessed by using the proper {id}, in this case the paging is not require. A place can't have an infinite number of connections to other places.

Also a single vehicle can be accessed directly. The suggested path can become really huge in relation to a wolrd-scale system. The retriving of the path is also divided in pages, the first page is always included, the others can be accessed separately.

Al kind of classes make use of link to other rescources to simplify the use of the APIs, also the ID of the stuffs (Plate or Place) is available.

The system use the basic HTTP Error Code:
ErrorTypeUsage
200OKCorrect execution
401Not AuthorizedUsed when the request require the admin option
403ForbiddenUsed when some request can't be accepted due to logical error (GateType)
404Not FoundResource asked not founded
409ConflictTrying to add a resource already available (ID) in the system
422Unprocessable EntityRetrived when some vehicle actions make use of not correct places or other similar stuff
